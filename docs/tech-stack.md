# پیشنهاد پشته فناوری (Tech Stack)

هدف: فروشگاه چندمنظوره با سرعت بالا، پنل مدیریت بدون‌کد، اتصال به درگاه‌های پرداخت ایرانی (زرین‌پال/پی‌پینگ)، ورود اجتماعی، باشگاه مشتریان و پیشنهاد محصول.

## ۱. فرانت‌اند (وب‌سایت مشتری)
| بخش | فناوری | دلیل انتخاب |
|---|---|---|
| فریم‌ورک | **Next.js 14+ (App Router) + TypeScript** | SSR/ISR برای سئو و سرعت بارگذاری بالا، Image Optimization داخلی |
| استایل | **Tailwind CSS** + پشتیبانی کامل RTL | توسعه سریع، باندل کوچک، سازگار با فارسی |
| مدیریت state | React Query (TanStack Query) + Zustand | کش داده سرور + state سبک برای سبد خرید/UI |
| فرم‌ها | React Hook Form + Zod | اعتبارسنجی سمت کلاینت هماهنگ با API |
| فونت | Vazirmatn (فونت فارسی متن‌باز) | خوانایی و ظاهر حرفه‌ای فارسی |

## ۲. بک‌اند
| بخش | فناوری | دلیل انتخاب |
|---|---|---|
| API | **NestJS (Node.js/TypeScript)** یا Next.js API Routes برای MVP | معماری ماژولار، مناسب رشد به میکروسرویس |
| دیتابیس اصلی | **PostgreSQL** | تراکنش‌محور، مناسب سفارش/پرداخت/موجودی |
| ORM | Prisma | Type-safe، migration ساده |
| کش/صف | **Redis** + BullMQ | کش محصولات پربازدید، صف پردازش امتیاز/ایمیل/پیامک |
| جستجو | **Meilisearch** (یا Elasticsearch در مقیاس بزرگ‌تر) | جستجوی فارسی سریع با تحمل غلط املایی |
| فایل/تصویر | S3-compatible Object Storage (آروان کلود یا لیارا) + CDN | بارگذاری سریع تصاویر محصول |

## ۳. پنل مدیریت (بدون نیاز به برنامه‌نویسی)
- **گزینه پیشنهادی:** ساخت پنل اختصاصی با **React Admin** یا **Refine.dev** روی همان API — امکان drag & drop برای تصاویر (react-dropzone) و جدول‌های قابل ویرایش inline.
- **جایگزین سریع‌تر برای MVP:** استفاده از **Payload CMS** یا **Strapi** به‌عنوان لایه مدیریت محتوا/محصول که به‌صورت headless به فرانت Next.js متصل می‌شود؛ این‌ها به‌صورت پیش‌فرض فرم‌های گرافیکی، آپلود فایل و مدیریت نقش/دسترسی دارند.
- سطوح دسترسی (RBAC) روی نقش‌ها: مدیر کل، مدیر محصول، پشتیبانی، انباردار.

## ۴. احراز هویت و ورود اجتماعی
- **NextAuth.js (Auth.js)** با provider های Google، و OAuth سفارشی برای تلگرام (Telogram Login Widget) و اینستاگرام (Instagram Basic Display/Graph API).
- JWT + Refresh Token برای سشن‌های API.

## ۵. پرداخت
- ماژول Payment Gateway انتزاعی (Adapter pattern) با پیاده‌سازی برای:
  - **زرین‌پال** (اصلی)
  - **پی‌پینگ** به‌عنوان بک‌آپ
  - کیف پول داخلی (موجودی حساب کاربر در دیتابیس با تراکنش‌های ledger-style)
  - گزینه پرداخت در محل (COD) برای سفارش‌های حضوری
- تمام تراکنش‌ها روی HTTPS/TLS و اطلاعات کارت هرگز روی سرور ذخیره نمی‌شود (توکن‌سازی توسط درگاه).

## ۶. باشگاه مشتریان و پیشنهاد محصول
- محاسبه امتیاز و سطح مشتری به‌صورت **job async** بعد از تکمیل سفارش (BullMQ worker).
- پیشنهاد محصول:
  - **MVP:** کوئری SQL "کاربران این را هم خریده‌اند" بر پایه هم‌رخدادی سفارش‌ها (co-purchase).
  - **مرحله بعد:** سرویس پیشنهاد جدا (Python + LightFM/Implicit) که به‌صورت دوره‌ای مدل را از روی تاریخچه بازدید/خرید آموزش می‌دهد و از طریق یک API داخلی نتایج را برمی‌گرداند.

## ۷. زیرساخت و DevOps
- **Docker + docker-compose** برای توسعه، **Kubernetes یا PaaS ایرانی (لیارا/آروان)** برای پروداکشن.
- CI/CD: GitHub Actions (تست، build، deploy خودکار).
- مانیتورینگ: Sentry (خطاها) + Grafana/Prometheus یا سرویس مانیتورینگ میزبان.
- CDN برای تصاویر و فایل‌های استاتیک جهت سرعت بارگذاری بالا.

## ۸. جمع‌بندی معماری
```
[Next.js Storefront] ---- REST/GraphQL ---- [NestJS API] ---- [PostgreSQL]
        |                                        |                |
     [CDN/Images]                            [Redis Queue]   [Meilisearch]
        |                                        |
[Admin Panel: React Admin/Payload]      [Workers: امتیاز، پیشنهاد، پیامک/ایمیل]
        |
[Payment Adapters: زرین‌پال/پی‌پینگ]      [Social Login: Google/Telegram/Instagram]
```
